# 데이터베이스
- 통합하여 관리되는 데이터의 집합
## DBMS 
- 데이터베이스를 관리하기 위한 `소프트웨어`

- 관계형 데이터 베이스 (RDBMS)
    -
- 데이터를 모아둔 테이블이 서로 관계를 맺고 모여있는 집합체

## DML
- 데이터 조회 및 변경하기 위한 명령어

- DML의 종류
    -
1. 조회 SELECT (셀렉트)(테이블 조회)
2. 삽입 INSERT (인서ㅌ)
3. 삭제 DELETE (딜릿ㅌ)
4. 변경 UPDATE

## 연산자
    -
1. 산술연산자(*,/,+,-)
2. 연결연산자(||)
3. 비교연산자
4. IS NULL, NULL , LIKE , IN
5. BETWEEN AND
6. NOT(논리부정연산자)
7. AND
8. OR

### ---
- NVL (오라클 전용) 계산 문제로 출제기능: 
- 첫 번째 값이 NULL이면 두 번째 값으로 바꿉니다.

- ISNULL (SQL Server 전용) ➡️ 짝짓기 문제로 출제기능: NVL과 완벽히 똑같습니다. (첫 번째 값이 NULL이면 두 번째 값 반환)시험 유형: "오라클의 NVL 함수와 기능이 같은 SQL Server의 함수는 무엇인가?" 또는 보기 중에 NVL(컬럼, 0)과 ISNULL(컬럼, 0)을 섞어놓고 올바른 문법을 고르는 문제로 출제됩니다.3. COALESCE (표준 SQL) ➡️ 고난도 다중 인자 문제로 출제 (★ 변별력 최고)기능: 나열된 인자 중에서 NULL이 아닌 첫 번째 값을 찾아 반환합니다.시험 유형: 인자를 2개만 주지 않고 3~4개씩 길게 줍니다.예시: COALESCE(NULL, NULL, '사과', '바나나') ➡️ 정답은 앞의 NULL들을 건너뛰고 처음으로 NULL이 아닌 '사과'가 됩니다.이 성질을 이용해 여러 컬럼을 순서대로 검사하는 고난도 쿼리 해석 문제로 자주 나옵니다.💡 하나 더! 시험에 같이 나오는 NULLIF이 세 개와 함께 NULLIF(A, B)라는 함수도 선택지에 무조건 껴서 나옵니다.기능: A와 B가 같으면 NULL을 남기고, 다르면 A를 남기는 함수입니다.COALESCE나 NVL과 겉모습이 비슷해서 보기 4개 중 하나로 무조건 출제되니 헷갈리면 안 됩니다.

## 별칭
    ```SQL
    SELECT SALARY*12 AS ANNUAL_SALARY FROM EMPLOYEES;
    -- AS 생략 가능하다 , 별칭으로 "월급" 한글도 사용 가능함
    ```
## DISTINCT(디스틴ㅌ)
    - 중복을 제거하고 조회 한다
    ```SQL
    SELECT DISTINCT MANAGER_ID FROM EMPLOYEES;
    ```
